## Objectif

Exécuter `scripts/train_unsupervised.py` pour les 3 modèles non supervisés (K-Means, KNN, PCA) sur Dataset 2 en scénario domain-incremental (Pump → Turbine → Compressor) et collecter les métriques CL complètes.

Cette expérience est le pendant non supervisé de `exp_001` (EWC) et `exp_002` (HDC) sur Dataset 2. Ses résultats alimenteront directement le notebook comparatif S5-09.

**Critère de succès** :
- `experiments/exp_005_unsupervised_dataset2/results/metrics_all.json` présent avec `aa`, `avg_forgetting`, `bwt`, `auroc_final`, `ram_peak_bytes`, `inference_latency_ms` pour les 3 modèles
- Aucun modèle ne dépasse `ram_peak_bytes > 65536` (64 Ko)
- `acc_matrix_*.npy` de shape `[3, 3]` avec NaN sur la partie supérieure

## Métriques attendues

| Métrique | KMeans | KNN | PCA | Remarque |
|---------|:------:|:---:|:---:|----------|
| `aa` | > 0.70 | > 0.70 | > 0.65 | Sans supervision — inférieur à EWC (0.9824) |
| `avg_forgetting` | ≈ 0.0 | ≈ 0.0 | variable | KMeans/KNN ne réécrivent pas les données passées |
| `bwt` | ≥ 0.0 | ≥ 0.0 | variable | |
| `auroc_final` | > 0.65 | > 0.65 | > 0.60 | Dataset 2 a peu de domain shift |
| `ram_peak_bytes` | < 8 192 | < 32 768 | < 4 096 | KNN stocke X_ref → plus grand |
| `inference_latency_ms` | < 1.0 | < 5.0 | < 1.0 | Opérations sklearn sur 4 features |

> **Si `aa < 0.50` pour tous les modèles** : vérifier que `anomaly_percentile` dans le YAML est adapté au ratio faulty/normal du Dataset 2. Un déséquilibre fort (ex. 5 % de faulty) nécessite un percentile > 95.
>
> **Si `auroc_final ≈ 0.50`** : les modèles ne discriminent pas — le domain shift du Dataset 2 est insuffisant pour les baselines non supervisées. Documenter comme limitation dans `docs/triple_gap.md` (Gap 1).

## Sous-tâches de collecte et vérification

### 1. Vérifier la sortie après exécution

```python
# Vérification post-run (à exécuter manuellement ou via pytest)
import json
import numpy as np
from pathlib import Path

exp_dir = Path("experiments/exp_005_unsupervised_dataset2/results")

# Vérifier metrics_all.json
with open(exp_dir / "metrics_all.json") as f:
    metrics_all = json.load(f)

REQUIRED_KEYS = ["acc_final", "avg_forgetting", "backward_transfer",
                 "auroc_final", "ram_peak_bytes", "inference_latency_ms", "n_params"]
for model_name, m in metrics_all.items():
    missing = [k for k in REQUIRED_KEYS if k not in m]
    assert not missing, f"{model_name} manque : {missing}"
    assert m["ram_peak_bytes"] <= 65536, f"{model_name} dépasse 64 Ko !"

# Vérifier shape acc matrices
for model_name in ["kmeans", "knn", "pca"]:
    mat = np.load(exp_dir / f"acc_matrix_{model_name}.npy")
    assert mat.shape == (3, 3), f"Shape inattendue : {mat.shape}"
    assert np.isnan(mat[0, 1]), "Case future non NaN"

print("Vérification exp_005 : OK")
```

## Diagnostics fréquents

| Symptôme | Cause probable | Solution |
|----------|----------------|----------|
| `acc_final < 0.50` tous modèles | Déséquilibre classes / seuil inadapté | Ajuster `anomaly_percentile` dans YAML |
| `auroc_final ≈ 0.50` | Peu de domain shift Dataset 2 | Documenter comme limitation Gap 1 |
| `KNN ram_peak_bytes > 65536` | `n_neighbors_max` trop grand | Réduire dans `unsupervised_config.yaml` |
| `FileNotFoundError: normalizer_path` | Normalizer non généré | Relancer S1-03 avec save_normalizer=True |
| `ValueError: roc_auc_score` | Val set monoclasse | Vérifier split — augmenter `val_ratio` |
