## Objectif

Étendre `scripts/train_unsupervised.py` pour supporter Dataset 1 (Pump Maintenance — séries temporelles multivariées) et exécuter l'expérience `exp_006` sur les 3 mêmes modèles non supervisés (K-Means, KNN, PCA) en scénario temporal domain-incremental.

**Différence clé avec exp_005** : Dataset 1 est une série temporelle. Les 3 tâches sont des fenêtres temporelles successives représentant l'évolution de la pompe (saine → usure naissante → pré-panne). Les features sont extraites via `PumpMaintenanceDataset.extract_features()` (S3-02). Il n'y a pas de domaine discret — le drift est continu et progressif.

**Hypothèse scientifique** : La détection d'anomalie non supervisée doit être plus naturelle sur Dataset 1 que sur Dataset 2 (drift plus prononcé entre tâches), ce qui devrait se traduire par un `auroc_final` plus élevé.

**Critère de succès** :
- `experiments/exp_006_unsupervised_dataset1/results/metrics_all.json` présent
- `auroc_final > 0.70` pour au moins 1 modèle (drift temporel exploitable)
- Shape `acc_matrix_*.npy` : `[3, 3]`

### Loader Dataset 1 (`load_pump_tasks`)

```python
def load_pump_tasks(config: dict) -> list[dict]:
    """
    Charge Dataset 1 (Pump) en scénario domain-incremental temporel.

    Retourne le même format que get_cl_dataloaders() de monitoring_dataset.py :
    list de dicts avec keys : train_loader, val_loader, domain, n_train, n_val.

    Protocole temporel (même que S3-06 / exp_003) :
      - Tâche 1 : t = 0..T/3      (pompe saine)
      - Tâche 2 : t = T/3..2T/3  (usure naissante)
      - Tâche 3 : t = 2T/3..T    (pré-panne)

    Parameters
    ----------
    config : dict
        Config complète (unsupervised_config.yaml, section "data_pump").

    Returns
    -------
    list[dict]
        [{"train_loader": ..., "val_loader": ..., "domain": "pump_t1", "n_train": int, "n_val": int}, ...]
    """
    from src.data.pump_dataset import PumpMaintenanceDataset, CLStreamSplitter
    from torch.utils.data import DataLoader, TensorDataset
    import torch

    data_cfg = config["data_pump"]
    dataset = PumpMaintenanceDataset(data_cfg["csv_path"])
    features, labels = dataset.extract_features(
        window_size=data_cfg.get("window_size", 50),
        step_size=data_cfg.get("step_size", 10),
    )
    cl_stream = CLStreamSplitter(
        features, labels,
        n_tasks=data_cfg.get("n_tasks", 3),
        strategy=data_cfg.get("task_strategy", "temporal"),
    )
    normalizer_path = data_cfg.get("normalizer_path")
    if normalizer_path:
        cl_stream.load_normalizer(normalizer_path)
        cl_stream.apply_normalizer(cl_stream._normalizer)

    domain_names = ["pump_healthy", "pump_wearing", "pump_prefailure"]
    tasks = []
    val_ratio = data_cfg.get("val_ratio", 0.2)
    batch_size = data_cfg.get("batch_size", 256)
    seed = config.get("seed", 42)

    for task_id in range(cl_stream.n_tasks):
        task_features, task_labels = cl_stream.get_task(task_id)
        n = len(task_labels)
        n_val = max(1, int(n * val_ratio))
        n_train = n - n_val

        # Split chronologique (pas de shuffle — série temporelle)
        X_train = torch.tensor(task_features[:n_train], dtype=torch.float32)
        y_train = torch.tensor(task_labels[:n_train], dtype=torch.float32)
        X_val   = torch.tensor(task_features[n_train:], dtype=torch.float32)
        y_val   = torch.tensor(task_labels[n_train:],   dtype=torch.float32)

        train_loader = DataLoader(
            TensorDataset(X_train, y_train),
            batch_size=batch_size,
            shuffle=False,   # Conserver l'ordre temporel
        )
        val_loader = DataLoader(
            TensorDataset(X_val, y_val),
            batch_size=batch_size,
            shuffle=False,
        )
        tasks.append({
            "train_loader": train_loader,
            "val_loader":   val_loader,
            "domain":       domain_names[task_id],
            "n_train":      n_train,
            "n_val":        n_val,
        })

    return tasks
```

## Métriques attendues

| Métrique | KMeans | KNN | PCA | Remarque |
|---------|:------:|:---:|:---:|----------|
| `aa` | > 0.65 | > 0.65 | > 0.60 | Drift progressif — seuil à recalibrer |
| `avg_forgetting` | variable | variable | variable | Drift temporel ≠ domaines discrets |
| `auroc_final` | > 0.70 | > 0.70 | > 0.65 | Dataset 1 a un drift plus net que Dataset 2 |
| `ram_peak_bytes` | < 32 768 | < 65 536 | < 8 192 | KNN stocke plus de références (~50 features) |
| `inference_latency_ms` | < 2.0 | < 10.0 | < 2.0 | ~50 features vs 4 pour Dataset 2 |

> **Si `auroc_final < 0.55`** pour tous les modèles : le drift temporel du Dataset 1 est peut-être moins prononcé que prévu en termes de score d'anomalie. Vérifier la distribution des labels par tâche (`y.mean()` par tâche).
>
> **Si `avg_forgetting > 0.10`** pour KMeans : la stratégie `fit_task` réinitialise-t-elle les centroides ? Vérifier `kmeans_config.yaml: cl_strategy`. Utiliser `accumulate` si le forgetting est problématique.
